# I18n — Đa ngôn ngữ EN / ZH / VI — Design Spec
Date: 2026-06-11

## Overview

Thêm hỗ trợ 3 ngôn ngữ (Tiếng Anh, Tiếng Trung giản thể, Tiếng Việt) cho toàn bộ giao diện app học tiếng Trung. Ngôn ngữ mặc định detect từ `navigator.language`, user có thể đổi bất cứ lúc nào qua dropdown trong navbar, lựa chọn lưu vào `localStorage`.

## Thư viện

`vue-i18n@9` — chuẩn công nghiệp cho Vue 3. Cài: `npm install vue-i18n@9`.

## Cấu trúc file mới

```
frontend/src/i18n/
  index.js        ← khởi tạo createI18n, detect ngôn ngữ, export instance
  vi.js           ← tất cả chuỗi tiếng Việt (nguồn gốc)
  en.js           ← tiếng Anh
  zh.js           ← tiếng Trung giản thể

frontend/src/components/
  LanguageSwitcher.vue   ← icon 🌐 + dropdown chọn ngôn ngữ
```

## Logic detect ngôn ngữ mặc định (`i18n/index.js`)

```js
const saved = localStorage.getItem('locale')
const browser = navigator.language?.toLowerCase() || ''
const detected = saved
  || (browser.startsWith('zh') ? 'zh' : browser.startsWith('vi') ? 'vi' : 'en')
```

Thứ tự ưu tiên: `localStorage` → browser language → fallback `'en'`.

## `LanguageSwitcher.vue`

- Nằm trong navbar `App.vue`, đặt sau `.btn-logout` / `.btn-login`
- Template: button `🌐 {tên ngắn} ▾` → click toggle dropdown
- Dropdown 3 mục: `🇺🇸 English`, `🇨🇳 中文`, `🇻🇳 Tiếng Việt` — mục đang active có dấu ✓
- Khi chọn: dùng `const { locale } = useI18n()` rồi set `locale.value = code` + `localStorage.setItem('locale', code)`
- Đóng dropdown khi click ra ngoài (click-outside listener)

## Cách dùng trong component

```vue
<script setup>
import { useI18n } from 'vue-i18n'
const { t } = useI18n()
</script>

<template>
  <h1>{{ t('home.title') }}</h1>
  <button>{{ t('auth.login') }}</button>
</template>
```

```js
export default {
  nav: { home: 'Trang chủ', journey: 'Hành trình', ... },
  auth: { login: 'Đăng nhập', password: 'Mật khẩu', ... },
```

`en.js` và `zh.js` có cùng cấu trúc key, khác value.

## Constraints

- Không có key nào undefined ở runtime — `en.js` là fallback locale, phải có đủ 100% keys
- `vue-i18n` dùng `legacy: false` (Composition API mode)
- Không dùng `$t()` trong `<script setup>` — dùng `const { t } = useI18n()` thay thế
- Ngôn ngữ Trung dùng giản thể (Simplified Chinese), không phải phồn thể
